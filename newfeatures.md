Offer Prep Tool — Финальное видение продукта

1. Что такое Offer Prep Tool
   Offer Prep Tool — внутренний веб-инструмент для подготовки HTML-офферов к деплою на трекинговую систему Keitaro. Цель финальной версии — сделать так, чтобы любой член команды, без технических знаний, мог самостоятельно взять сырую папку с оффером и за несколько минут получить готовую ссылку на оффер в кейтаро.

2. Принципы продукта
   Принцип

Описание

Zero knowledge

Шаг 1 — Upload (Загрузка оффера)
Без изменений относительно текущей версии. Пользователь перетаскивает или выбирает папку с оффером.

Что происходит автоматически:

Поиск index.html в дереве папок

Нормализация структуры файлов (js/, css/, img/, fonts/)

Обновление всех путей (src, href, url(), @import)

Форматирование HTML через Prettier

Добавление инклюдов формы и всего что нужно для нашей формы

2.2 — iFrame
Автоматическое обнаружение всех <iframe> тегов в HTML

Отображение списка с preview URL и размером

Возможность удалить каждый iFrame по отдельности или все сразу

2.3 — Неиспользуемые файлы
Сканирование всех файлов сессии

Удаление автоматически

Результат шага: чистый оффер без трекинга, лишних iFrame и мусорных файлов.

Шаг 3 — Content Editor (Редактирование контента)
Визуальный сервис редактирования контента оффера — аналог простой CMS-админки. Состоит из трёх инструментов:

3.1 — Live Text Editor
Интерфейс: слева — живой визуальный превью страницы, справа — панель редактирования

Клик по любому тексту в превью → поле редактирования справа с тегом и текущим значением

Изменения отражаются в превью мгновенно (без перезагрузки)

Поддержка редактирования: заголовки, параграфы, кнопки, списки

3.2 — Bulk Text Replace
Загрузка документа с текстом.

Инструмент который самостоятельно по абзацам и предложениям вставляет текст согласно порядку в документе

Подтверждение → массовая замена одним кликом

3.3 — Image Manager
Полный менеджер изображений оффера:

Функция

Описание

Галерея всех изображений оффера с именем файла, размером и тегом в HTML

Результат шага: оффер с нужным контентом, правильными текстами и оптимизированными изображениями.

4. Экстренное вмешательство (Dev Access)
   Механизм помощи разработчика пользователю без доступа к его машине.

Как это работает
Пользователь в любой момент может нажать "Share Session" в хедере

Инструмент показывает Session ID (например: ses_a3f9k2)

Пользователь передаёт этот ID разработчику (в Slack, Telegram и т.д.)

Разработчик открывает специальный URL: http://tool.internal/dev?session=ses_a3f9k2

Он видит полное состояние сессии: дерево файлов, лог всех действий, текущий шаг

Что может делать разработчик в режиме Dev Access
Возможность

Описание

Просмотр файлов

Полный файловый браузер сессии

Редактирование файлов

Встроенный редактор (CodeMirror) для прямого редактирования HTML/CSS/JS/PHP

Просмотр лога

Хронологический лог всех действий пользователя в сессии

Принудительный шаг

Возможность перевести сессию на любой шаг

Скачивание

Скачать текущий снимок сессии как ZIP в любой момент

Сессия доступна по ID до тех пор, пока она не удалена вручную или не истёк TTL (время жизни).

5. Управление сессиями
