## 1. Огляд проекту

**Offer Prep Tool** — веб-застосунок для підготовки HTML-лендінгів до деплою на Keitaro. Має два режими роботи: **Standard** (для баєрів і копірайтерів без технічного досвіду) і **Developer** (для розробників з повним контролем над файлами).

### Що робить інструмент

- Нормалізує структуру файлів до стандартного вигляду (`js/`, `css/`, `img/`, `fonts/`)
- Оновлює всі шляхи (`src`, `href`, `url()`, `@import`)
- Автоматично очищає трекінгові скрипти (Google Analytics, GTM, Facebook Pixel, Matomo тощо)
- Дозволяє редагувати текст, замінювати зображення та відео через інтерактивне превʼю
- Підтримує видалення будь-якого елемента DOM з інтерактивного превʼю (pick-to-delete)
- Вставляє готові елементи та віджети після будь-якого елемента сторінки
- Інтегрує PHP-сніпети для Keitaro, генерує `send.php`
- Надає Monaco-редактор для ручного втручання в код (Dev режим)
- Збирає готовий ZIP-архів

## 5. Режими роботи

На головній сторінці (`/`) — вибір режиму.

### Standard (`/standard`) — для баєрів і копірайтерів

**Pipeline:** Upload → Content → PHP → Build

- При завантаженні автоматично виконується **auto-clean** — без ручного втручання
- Сервер зберігає `_originals/` — snapshot файлів до очищення, щоб дев міг запустити повний pipeline з нуля
- Polling кожні 3с: якщо дев щось змінив у сесії — показується нотифікація `"Developer updated the session ↺"`

### Dev (`/dev`) — для розробників

**Pipeline:** Upload → Clean → Content → PHP → Build

- Повний контроль над кожним кроком
- Вкладка **"💻 Code Editor"** у Content Editor: Monaco з Emmet, дерево файлів, превʼю у новій вкладці
- Кнопка **"Share Session"** → URL для Dev Access
- Список всіх сесій: `/dev?sessions=1`

## 6. Pipeline — Standard режим

### Крок 1 — Upload

Перетягніть або оберіть папку з офером.

**Що відбувається на сервері:**
1. `multer` приймає файли в памʼять (`memoryStorage`)
2. Шляхи відновлюються з поля `filePaths` (JSON) — multer v2 стирає слеші з `originalname`
3. Файли записуються в `sessions/<id>/raw/` зі збереженням структури
4. `folderScanner` знаходить `index.html` за мінімальною глибиною
5. `fileNormalizer` нормалізує структуру та оновлює всі посилання
6. HTML форматується через Prettier
7. Зберігається `_originals/` (snapshot для дева)
8. `raw/` видаляється
9. Автоматично запускається **auto-clean** (скрипти + iframes + unused files)

**Структура після нормалізації:**
```
sessions/<id>/
├── index.html
├── js/
├── css/
├── fonts/
└── img/
```

### Крок 2 — Clean (тільки Dev)

Чотири вкладки:

#### Scripts

Класифікація та вибіркове видалення всіх `<script>` і `<noscript>` тегів.

**Категорії скриптів:**

| Категорія | Значок | Логіка |
|---|---|---|
| Tracking | 🔴 | Відомі трекінгові патерни: GA, GTM, FB Pixel, Hotjar, Matomo, `_paq`, `advertising.js` тощо |
| External Unknown | 🔴 Ext Script | Зовнішній CDN-скрипт, який не потрапив в Critical/FormLib — видаляється як потенційний трекер |
| Interactive | 🟡 | Маніпулює DOM (`classList.add/remove/toggle`, `innerHTML` тощо) — потребує уваги |
| Form Handler | 🟡 | Обробники форм, submit-логіка |
| Critical | 🟢 | jquery, bootstrap, vue, react, main.js, lazysizes, picturefill тощо |
| Noscript | ⚪ noscript | `<noscript>` теги — зазвичай трекінгові пікселі |
| Unknown | ⚪ | Решта скриптів |

**Авто-відмічені на видалення (pre-checked):**
- Tracking
- External Unknown
- Noscript
- Локальні `.js` файли, що складаються повністю з коментарів (`isEffectivelyEmpty`)

**Auto-clean (Standard режим)** видаляє автоматично всі скрипти категорій Tracking та External Unknown без показу інтерфейсу.

**Деталі класифікації (`htmlProcessor.js`):**

```
TRACKING_PATTERNS       — src-патерни: googletagmanager, analytics, fbevents,
                          hotjar, matomo.js, advertising.js тощо
INLINE_TRACKING_PATTERNS — inline-патерни: gtag(, _paq.push, _paqM.push,
                          fbq(, dataLayer.push тощо
CRITICAL_PATTERNS       — src-патерни: jquery, bootstrap, vue.min, react,
                          main.js, lazysizes, picturefill тощо
FORM_LIB_PATTERNS       — src-патерни: form, validate, submit тощо
DOM_MUTATION_PATTERNS   — inline: classList.add/remove/toggle/replace,
                          innerHTML, insertAdjacentHTML тощо
```

#### iFrames

Список усіх `<iframe>` тегів — src, розміри, атрибути. Вибіркове видалення.

#### Unused Files

Файли в папках `js/`, `css/`, `img/`, `fonts/`, на які немає жодного посилання в HTML або CSS. Вибіркове видалення.

#### Head Items

`<link>`, `<meta>`, `<style>` та інші теги з `<head>`. Вибіркове видалення (наприклад, зайві `preconnect`, `dns-prefetch` до трекінгових доменів).

### Крок 3 — Content Editor

Чотири вкладки:

#### ✏️ Live Text Editor

Iframe-превʼю офера зліва, панель редагування справа.

**Режими превʼю (перемикаються кнопками в toolbar):**

| Кнопка | Режим | Деталі |
|---|---|---|
| 🖥 Desktop | 1280px scaled | Iframe рендериться при 1280px, масштабується через `transform: scale()` щоб влізти в пане. Активний за замовчуванням. |
| 📱 Mobile | 375px | Iframe 375px, відцентрований. Натиснути ще раз — повернення до Desktop. |

**Інтерактивна редакція тексту:**
- Клік на `h1–h6`, `p`, `button`, `a`, `label`, `li` → відкриває textarea з поточним innerHTML
- Live-update через `postMessage` — зміни видно в iframe одразу, без збереження
- Кнопка **Save** — зберігає через cheerio в `index.html`
- Кнопка **Discard** — скасовує незбережені зміни
- Кнопка **✦ Format** — форматує HTML сніпет через Prettier

**Редакція зображень:**
- Клік на `<img>` → панель із превʼю, розмірами, кнопкою заміни файлу
- Кнопка **↑ Replace Image** — завантажує новий файл, оновлює в iframe без перезавантаження
- Кнопка **✕ Delete** — видаляє `<img>` з DOM (з підтвердженням)
- Кнопки **⊕ Add element** / **⊞ Add widget** — вставити елемент/віджет після цього зображення

**Редакція відео:**
- Клік на `<video>` → панель з постером, кнопкою заміни
- **↑ Replace Video** — завантажує відео, автоматично генерує WebP-постер із першого кадру, додає атрибут `controls`
- Кнопка **✕ Delete** та **⊕/⊞ Add** — аналогічно зображенням

**Pick-to-delete mode (🗑 Delete):**

Режим вибіркового видалення будь-якого елемента DOM.

| Дія | Результат |
|---|---|
| Натиснути `🗑 Delete` | Вмикається pick mode. Приховані елементи (`display:none`) стають видимими з помаранчевим обводом |
| Hover | Елемент підсвічується червоним пунктиром |
| Single click | Автоматично вибирається найближчий значущий батьківський блок: спочатку semantic (`aside`, `section`, `article`, `nav`, `header`, `footer`, `form`), потім перший елемент з class/id |
| Double click | Вибирається точний елемент під курсором (innermost) |
| Панель справа | Показує лейбл елемента, його HTML-превʼю, chips предків для навігації вгору |
| Клік на chip предка | Перемикає ціль видалення на батьківський елемент + підсвічує його в iframe |
| Кнопка `🗑 Delete` | Видаляє елемент через CSS-selector (`nth-child` path), зберігає в `index.html` |
| `↩ Undo` | З'являється після видалення. Відновлює попередній стан з `.undo` backup |
| `✕ Cancel` | Виходить з pick mode |

> **Як працює CSS-selector path:** `buildEptPath()` будує шлях `tag:nth-child(N)>tag:nth-child(N)>...` від кореня до елемента. Цей самий selector використовується і в cheerio на сервері для точного знаходження елемента.

**Insert after (⊕ Add element / ⊞ Add widget):**
- Доступно для text-елементів, зображень і відео
- Відкриває picker-модал з каталогом елементів (унікальні елементи з `<main>` з їх стилями) або з списком віджетів
- Вставляє після вибраного елемента через `afterIdx` (для text) або `afterSelector` CSS-path (для img/video)

#### 📋 Bulk Replace

- Завантажити `.txt` файл або вставити текст вручну
- Абзаци (розділені пустим рядком) → послідовна заміна editable-елементів
- Попередній перегляд маппінгу перед застосуванням

#### 🖼 Image Manager

- Галерея всіх зображень сесії з розмірами файлів
- Заміна файлу, конвертація формату (WebP / JPEG / PNG)
- Стиснення через sharp з налаштуванням якості
- **Compress All** — batch-конвертація всіх зображень одним запитом

#### 💻 Code Editor (тільки Dev mode)

- Дерево файлів проекту зліва (тільки текстові: html/css/js/php/json/svg тощо)
- **Monaco Editor** з підсвіткою синтаксису (автодетекція по розширенню)
- **Emmet** для HTML і CSS (`div.container>ul>li*3` → Tab)
- Кнопка **"Open Preview ↗"** — відкриває превʼю у новій вкладці
- Після **Save** — превʼю у тій вкладці перезавантажується через `BroadcastChannel`

- Копіює поточні файли dev-сесії назад у сесію баєра
- Оновлює `lastDevActivity` → баєр бачить нотифікацію
- Не перезаписує `_originals/`, `_activity_log.json`, `_session_meta.json`, `_offer_config.json`

```
├── Routes:
│   ├── /api/upload    → upload.js

│   └── /api/dev       → dev.js
├── GET /api/health    → { status: 'ok' }

```

### Preview iframe — injected editor script

`buildEditorScript(sid)` в `content.js` генерує JS який інжектується перед останнім `</body>`.

**Що робить скрипт:**

| Функція | Дія |
|---|---|
| Text click handlers | Реєструє click на `h1–h6,p,button,a,label,li` через `addEventListener(..., true)` |
| Image handlers | Реєструє click на всі `<img>` |
| Video handlers | Обгортає `<video>` в позиційований wrapper + transparent overlay div (блокує Safari autoplay) |
| `buildEptPath(el)` | Будує CSS nth-child path від кореня до елемента (використовується і на клієнті, і в cheerio) |
| `eptLabel(el)` | Короткий лейбл: `tag.class1.class2` |
| Catalog broadcast | Після завантаження збирає унікальні елементи з `<main>` + всі CSS посилання/inline стилі → `postMessage ept-catalog` |
| `setPickMode(on)` | Вмикає/вимикає pick-to-delete режим: показує приховані елементи, реєструє mouseover/click/dblclick handlers |
| `findMeaningfulBlock(el)` | При single-click піднімається вгору по DOM до першого semantic елемента або елемента з class/id |
| `sendPickMessage(el)` | Надсилає `ept-pick-delete` з selector, label, preview, ancestors[] (кожен з selector+label+preview) |

**postMessage API (iframe → parent):**

| Тип | Дані | Опис |
|---|---|---|
| `ept-catalog` | `items[], cssLinks[], inlineStyles[]` | Каталог елементів після завантаження |

| `ept-img-select` | `name, src, width, height, selectorPath` | Клік на зображення |
| `ept-video-select` | `name, src, poster, selectorPath` | Клік на відео |
| `ept-pick-delete` | `selector, label, preview, ancestors[]` | Клік в pick mode |

**postMessage API (parent → iframe):**

| Тип | Дані | Опис |
|---|---|---|

## 11. Сервіси

### `folderScanner.js`
Рекурсивний пошук `index.html`. Вибирає за мінімальною глибиною; при рівній — максимальна кількість асетів поруч (бонус за `js/`, `css/`, `assets/`).

### `fileNormalizer.js`
Нормалізує структуру та оновлює всі посилання: `<link href>`, `<script src>`, `<img src>`, `srcset`, `data-src`, `url()` у CSS, `@import` (рекурсивно), `<video src>`, `<source src>`, `<meta og:image>`.

### `htmlProcessor.js`

Ключові функції:

| Функція | Опис |
|---|---|
| `extractScripts()` | Класифікує всі `<script>` та `<noscript>` по багаторівневих патернах |
| `isEffectivelyEmpty(content)` | Видаляє `/* */` і `//` коментарі, перевіряє чи залишився код — якщо ні, скрипт вважається порожнім |
| `isExternalUnknown` | `true` якщо скрипт підключений з зовнішнього CDN і не відповідає Critical/FormLib патернам |
| `removeScripts()` | Видаляє скрипти по індексах |
| `injectSnippet()` | Вставляє HTML-сніпет в задану позицію |
| `detectInsertionPoints()` | Автовизначення точок вставки (after head, before/after form тощо) |
| `extractHeadItems()` | Витягує `<link>`, `<meta>`, `<style>` з `<head>` |
| `cleanHeadItems()` | Видаляє head items по індексах |

### `htmlFormatter.js`
Prettier 3 (async). Parser: `html`, printWidth: 120, tabWidth: 2. При помилці повертає оригінал з `success: false`.

### `widgetInjector.js`
Копіює файли віджета в сесію, додає `<link>` у `<head>`, вставляє HTML + `<script>` у задану позицію.

### `activityLogger.js`
Записує всі дії в `_activity_log.json` сесії з timestamp та деталями. Дії: `upload`, `auto-clean`, `clean-scripts`, `clean-iframes`, `clean-unused`, `clean-head`, `save-text`, `bulk-replace`, `replace-image`, `replace-video`, `insert-after`, `insert-widget`, `delete-element`, `delete-by-selector`, `undo-delete`, `php-apply`, `build`, `save-file`, `push-to-buyer`.

### `sendPhpGenerator.js`
Генерує `send.php` з `templates/send.php.template`, підставляючи `offerName`, `countryCode`, `langCode`.

## 12. Віджети

Кожен віджет — папка в `widgets/`:

```
widgets/<name>/
├── meta.json     # { "name": "...", "description": "..." }
├── widget.html
├── widget.css    # (опціональний)
└── widget.js     # (опціональний)
```

Щоб додати новий — створити папку з файлами. Сервер підбирає автоматично, перезапуск не потрібен.

**Де показуються:** Content Editor → кнопка "⊞ Add widget" → picker-модал з превʼю (HTML/JS/CSS/Preview вкладки) та кнопкою "⊕ Insert here".

## 13. Відомі обмеження

| Обмеження | Деталі |
|---|---|
| Макс. розмір файлу | 50 MB на файл (multer) |
| Макс. кількість файлів | 2 000 файлів на один upload |
| Тіло JSON запиту | 50 MB |
| Версія Node | ≥ 20.18.1 (cheerio 1.2) |
| Сесії | TTL 24г, автоочистка кожні 6г |
| Undo | Один крок назад. Тільки для delete операцій; зміни тексту не відміняються |
| `_originals/` | Зберігається тільки при Standard upload; для старих сесій — потрібен re-upload |
| Стиснення зображень | Потребує `sharp` (`npm install sharp` у `server/`) |
| PHP форматування | Prettier форматує `index.php`, але PHP-логіка не виконується |
| Emmet в Monaco | Підтримується для HTML і CSS; JS Emmet не підключено |
| BroadcastChannel | Не підтримується у Safari < 15.4 |
| Pick-to-delete | CSS nth-child selector може бути некоректним якщо сторінка динамічно змінює DOM після завантаження |
| Desktop preview scale | При дуже вузькому пані (< 400px) scale може бути надто малим |
