## 1. Огляд проекту

**Offer Prep Tool** — веб-застосунок із п'ятикроковим пайплайном обробки офер-папок (HTML-лендінгів). Після завантаження папки інструмент:

- нормалізує структуру файлів до стандартного вигляду
- автоматично оновлює всі шляхи (`src`, `href`, `url()`, `@import`)
- очищає трекінгові скрипти (Google Analytics, Facebook Pixel тощо)
- дозволяє вставити готові віджети в потрібне місце сторінки
- інтегрує PHP-сніпети для системи Keitaro
- генерує `send.php` з конфігурацією країни та мови
- форматує HTML через Prettier
- збирає готовий ZIP-архів для завантаження

## 4. Встановлення та запуск

### Вимоги

- **Node.js 20+** (cheerio 1.2 потребує Node ≥ 20.18)
- **nvm** (рекомендовано для управління версіями)

### Перший запуск

```bash
# 1. Переключитись на Node 20
nvm use 20

# Або встановити та зробити дефолтною назавжди
nvm install 20
nvm alias default 20

# 2. Встановити залежності
npm install
npm --prefix server install
npm --prefix client install

# 3. Запустити в режимі розробки
npm run dev
```

Скрипт `npm run dev` автоматично:
1. Перевіряє версію Node (< 20 → виходить з помилкою)
2. Вбиває процеси на портах `3001` і `5173` якщо вони зайняті
3. Запускає API-сервер і чекає поки він відповість на `/api/health`
4. Тільки після цього запускає UI

| Сервіс    | URL                       |
|-----------|---------------------------|
| UI        | http://localhost:5173     |
| API       | http://localhost:3001     |
| Health    | http://localhost:3001/api/health |

### Збірка для продакшену

```bash
npm run build    # будує React у client/dist/
npm run start    # запускає сервер + vite preview
```

## 5. Кроки роботи з інструментом

### Крок 1 — Upload (Завантаження папки)

Перетягніть або оберіть папку з офером. Підтримується вибір цілої директорії через `webkitdirectory`.

**Що відбувається на сервері:**

1. `multer` приймає файли в пам'ять (`memoryStorage`)
2. Шляхи файлів відновлюються з поля `filePaths` (JSON), бо multer v2 стирає слеші з `originalname`
3. Файли записуються в `sessions/<id>/raw/` зі збереженням структури
4. `folderScanner.findIndexHtml()` знаходить основний `index.html` (найменша глибина + найбільше асетів поруч)
5. `fileNormalizer.normalize()` нормалізує структуру
6. HTML форматується через Prettier
7. `raw/` видаляється

**Структура після нормалізації:**

```
sessions/<id>/
├── index.html      ← головний файл
├── js/             ← всі .js, .mjs
├── css/            ← всі .css (url() та @import оновлені)
├── fonts/          ← .woff, .woff2, .ttf, .eot, .otf
└── img/            ← .png, .jpg, .svg, .gif, .webp, .ico...
```

> Файли, на які немає посилань в HTML або CSS, видаляються автоматично.

**Нотифікація після завантаження:**
```
Found index.html · Moved 42 files · Removed 5 unused files · Auto-formatting successful
```

---

### Крок 2 — Script Cleaner (Очищення скриптів)

Сканує всі `<script>` теги в `index.html` і класифікує їх:

| Категорія  | Колір  | Логіка                                              |
|------------|--------|-----------------------------------------------------|
| Tracking   | 🔴 Red  | Збіг з патернами: GA, GTM, Facebook Pixel, Hotjar тощо |
| Critical   | 🟢 Green | Збіг з: jquery, bootstrap, vue, react, main.js тощо |
| Unknown    | ⚪ Gray  | Все інше                                            |

Tracking-скрипти автоматично відмічаються для видалення. Можна вручну скасувати або вибрати будь-які інші.

---

### Крок 3 — Widgets (Вставка віджетів)

Список доступних віджетів із `widgets/` директорії. Кожен віджет містить:

```
widgets/<name>/
├── meta.json    # { "name": "...", "description": "..." }
├── widget.html  # HTML-розмітка
├── widget.js    # JavaScript
└── widget.css   # Стилі
```

При вставці:
- JS і CSS копіюються в `sessions/<id>/widgets/<name>/`
- `<link>` тег додається в `<head>`
- HTML + `<script>` тег вставляються в обрану позицію

**Доступні позиції вставки:**

| ID                   | Опис                           |
|----------------------|--------------------------------|
| `before-body-close`  | Перед `</body>` (рекомендовано)|
| `after-body-open`    | Після відкриття `<body>`       |
| `before-head-close`  | Перед `</head>`                |
| `before-form-close`  | Всередині останньої `<form>`   |
| `selector:.<class>`  | Після конкретного CSS-селектора|

---

### Крок 4 — PHP Integration

Конфігурація параметрів офера для Keitaro.

| Поле          | Приклад      | Де використовується                                  |
|---------------|--------------|------------------------------------------------------|
| Offer Name    | `Quantum AI` | Hidden input `offer_name`, константа `OFFER_NAME`   |
| Country Code  | `DE`         | Константа `COUNTRY_CODE`                             |
| Language Code | `de`         | Атрибут `lang`, функція `getFormJSCss()`, `LANGUAGE_CODE` |

**PHP сніпети, що вставляються в `index.php`:**

```
Перед <!DOCTYPE>   → require_once global_new.php
Після <head>       → require_once google_event.php
Перед </head>      → echo getFormJSCss('de')
Перед </body>      → require_once offer_footer_script.php
Після <form>       → hidden input offer_name + require_once hidden_params.php
```

Всі ін'єкції **ідемпотентні** — повторний Apply не дублює сніпети.

Додатково генерується `send.php` з усіма параметрами (`COUNTRY_CODE`, `LANGUAGE_CODE`, `OFFER_NAME` тощо).

Вкладка **"send.php Preview"** показує фінальний вміст файлу в реальному часі відповідно до поточних значень форми.

---

### Крок 5 — Build Offer (Збірка)

Показує дерево файлів сесії. Кнопка **Download ZIP** упаковує всю сесію в архів (`offer-<id>.zip`, рівень стиснення 9).

Із архіву виключаються:
- файли з префіксом `_` (внутрішні: `_offer_config.json`)
- папка `raw/` (тимчасова)

## 6. Архітектура сервера

```
index.js
├── Middleware: cors, express.json (50mb), express.urlencoded (50mb)
├── Static: /widgets → ./widgets/
├── Routes:
│   ├── /api/upload   → upload.js
│   ├── /api/process  → process.js
│   ├── /api/widgets  → widgets.js
│   ├── /api/php      → php.js
│   └── /api/build    → build.js
├── GET /api/health   → { status: 'ok' }
└── Global error handler → 500 JSON
```

**Сесії** зберігаються в `server/sessions/<sessionId>/` на час роботи. Один `sessionId` генерується на фронтенді при кожному відкритті сторінки (`Math.random` + `Date.now`).

**nodemon** стежить тільки за: `index.js`, `routes/`, `services/`, `templates/`. Директорія `sessions/` виключена — щоб уникнути рестарту під час запису файлів при upload.

## 7. API довідник

### Upload

| Метод  | URL             | Опис                          |
|--------|-----------------|-------------------------------|
| POST   | `/api/upload`   | Завантажити папку офера       |
| DELETE | `/api/upload/:id` | Видалити сесію              |

**POST `/api/upload`**
- Headers: `X-Session-Id: <sessionId>`
- Body: `multipart/form-data` — поле `files[]` + поле `filePaths` (JSON-рядок)
- Response:
```json
{
  "sessionId": "abc123",
  "filesUploaded": 45,
  "indexPath": "index.html",
  "indexSizeKb": 128,
  "formatted": true,
  "normalization": { "moved": 42, "removed": 3, "warnings": [] }
}
```

## 8. Сервіси

### `folderScanner.js`
Рекурсивний пошук `index.html` у дереві папок. Стратегія вибору:
1. Мінімальна глибина вкладеності
2. При рівній глибині — максимальна кількість асетів у тій самій директорії (JS/CSS/зображення + бонус за стандартні папки `assets/`, `js/`, `css/`)

### `fileNormalizer.js`
Основний сервіс нормалізації. Обробляє:
- `<link href>` (CSS, preload)
- `<script src>` (JS)
- `<img src>`, `srcset`, `data-src`, `data-bg` тощо
- `<video poster>`, `<source src>`
- `[style]` з `url()`
- `<meta og:image>`
- `url()` у CSS файлах (з коригуванням відносного шляху `../`)
- `@import` у CSS файлах (рекурсивно, ітеративна черга)

### `htmlProcessor.js`
Робота зі скриптами та ін'єкціями:
- `extractScripts(html)` — класифікація по патернах
- `removeScripts(html, indices)` — видалення по індексах
- `injectSnippet(html, snippet, position)` — вставка в задану позицію
- `detectInsertionPoints(html)` — автоматичне визначення точок вставки

### `htmlFormatter.js`
Форматування через Prettier 3 (async API):
- Parser: `html`
- printWidth: 120, tabWidth: 2
- PHP-теги обробляються як opaque text — форматування зазвичай успішне навіть для `index.php`
- При будь-якій помилці повертає оригінальний HTML з `success: false`

### `widgetInjector.js`
- Копіює JS/CSS віджета в `sessions/<id>/widgets/<name>/`
- Додає `<link>` у `<head>`
- Вставляє HTML + `<script>` у задану позицію

### `sendPhpGenerator.js`
Генерує `send.php` з шаблону `templates/send.php.template`, підставляючи geo-конфігурацію як PHP-масив.

## 9. Віджети

Кожен віджет — окрема папка в `widgets/`:

```
widgets/
├── example-widget/        # Trust Badge Bar
│   ├── meta.json
│   ├── widget.html
│   ├── widget.css
│   └── widget.js
└── countdown-timer/       # Таймер зворотнього відліку
    ├── meta.json
    ├── widget.html
    ├── widget.css
    └── widget.js
```

**Щоб додати новий віджет** — створити папку в `widgets/` з файлами відповідно до структури вище. `meta.json` мінімум:
```json
{
  "name": "Назва віджета",
  "description": "Короткий опис"
}
```

Сервер підбирає всі папки автоматично — перезапуск не потрібен.

## 10. Відомі обмеження

| Обмеження                | Деталі                                                        |
|--------------------------|---------------------------------------------------------------|
| Макс. розмір файлу       | 100 MB на файл (налаштування multer)                          |
| Макс. кількість файлів   | 2 000 файлів на один upload                                   |
| Тіло JSON запиту         | 50 MB (express.json limit)                                    |
| Proxy timeout            | 120 секунд (vite proxy)                                       |
| Версія Node              | **≥ 20.18.1** (cheerio 1.2 вимога)                           |
| Сесії                    | Зберігаються локально, не очищаються автоматично              |
| PHP форматування         | Prettier форматує `index.php`, але PHP-логіка не виконується  |
| Drag & drop              | `webkitRelativePath` може бути відсутнім — рекомендується клік|
