## 1. Огляд проекту

**Offer Prep Tool** — веб-застосунок для підготовки HTML-лендінгів (офферів) до деплою на Keitaro. Має два режими роботи: **Standard** (для баєрів і копірайтерів) і **Developer** (для розробників з повним контролем над файлами).

**Що робить інструмент:**

- нормалізує структуру файлів до стандартного вигляду (`js/`, `css/`, `img/`, `fonts/`)
- оновлює всі шляхи (`src`, `href`, `url()`, `@import`)
- очищає трекінгові скрипти (Google Analytics, Facebook Pixel тощо)
- дозволяє редагувати текст, замінювати зображення та відео через інтерактивне превʼю
- вставляє готові віджети в потрібне місце сторінки
- інтегрує PHP-сніпети для Keitaro, генерує `send.php`
- надає Monaco-редактор для ручного втручання в код (Dev режим)
- збирає готовий ZIP-архів

## 5. Режими роботи

На головній сторінці (`/`) — вибір режиму:

### Standard (`/standard`) — для баєрів і копірайтерів
**Pipeline:** Upload → Content → PHP → Build

- При завантаженні автоматично виконується **auto-clean** (видалення трекінгових скриптів, iframe, невикористаних файлів) — без ручного втручання
- Сервер зберігає `_originals/` — snapshot файлів до очищення, щоб дев міг запустити повний pipeline з нуля
- Polling кожні 3с: якщо дев щось змінив у сесії — показується нотифікація "Developer updated the session ↺"

### Dev (`/dev`) — для розробників
**Pipeline:** Upload → Clean → Content → PHP → Build

- Повний контроль над кожним кроком
- Вкладка **"💻 Code Editor"** у Content Editor: Monaco з Emmet, дерево файлів, превʼю у новій вкладці
- Кнопка **"Share Session"** → URL для Dev Access
- Список всіх сесій: `/dev?sessions=1`

## 6. Pipeline — Standard режим

### Крок 1 — Upload

Перетягніть або оберіть папку з офером.

**На сервері:**
1. `multer` приймає файли в памʼять (`memoryStorage`)
2. Шляхи відновлюються з поля `filePaths` (JSON), бо multer v2 стирає слеші
3. Файли записуються в `sessions/<id>/raw/` зі збереженням структури
4. `folderScanner` знаходить `index.html`
5. `fileNormalizer` нормалізує структуру та оновлює всі посилання
6. HTML форматується через Prettier
7. Зберігається `_originals/` (snapshot для дева)
8. `raw/` видаляється
9. Автоматично запускається **auto-clean** (скрипти + iframes + unused files)

**Структура після нормалізації:**
```
sessions/<id>/
├── index.html
├── js/
├── css/
├── fonts/
└── img/
```

### Крок 2 — Content Editor

Три вкладки: **Live Text Editor**, **Bulk Replace**, **Image Manager** (детально — у розділі [Dev: Content Editor](#крок-3--content-editor-dev)).

### Крок 3 — PHP Integration

Детально — у розділі [PHP Integration](#крок-4--php-integration-dev).

### Крок 4 — Build

Download ZIP з файлами офера.

## 7. Pipeline — Dev режим

### Крок 1 — Upload

Аналогічно Standard, але **без** auto-clean і **без** збереження `_originals/` (дев і так стартує з нуля).

### Крок 2 — Clean (тільки Dev)

Три вкладки:

**Scripts** — класифікація всіх `<script>` тегів:

| Категорія | Логіка |
|---|---|
| 🔴 Tracking | GA, GTM, Facebook Pixel, Hotjar тощо — автоматично відмічені |
| 🟢 Critical | jquery, bootstrap, vue, react, main.js тощо |
| ⚪ Unknown | решта |

**iFrames** — список усіх `<iframe>`, вибіркове видалення.

**Unused Files** — файли, на які немає посилань в HTML/CSS — вибіркове видалення.

### Крок 3 — Content Editor (Dev)

Чотири вкладки:

#### ✏️ Live Text Editor
- Iframe-превʼю офера зліва, панель редагування справа
- Клік на елемент → textarea з поточним HTML, live update через postMessage
- Підтримувані елементи: `h1–h6`, `p`, `button`, `a`, `label`, `li`
- Клік на зображення → заміна файлу
- Клік на відео → заміна файлу (генерується poster)
- Кнопки: **Save** (зберігає через cheerio), **Insert after** (вставка нового елемента після виділеного), **Delete**
- Каталог елементів: унікальні елементи з `<main>` зі стилями — для вибору шаблону при вставці

#### 📋 Bulk Replace
- Завантажити `.txt` файл (абзаци = нові тексти по порядку)
- Масова заміна `editable elements` одним запитом

#### 🖼 Image Manager
- Галерея всіх зображень сесії
- Заміна файлу, конвертація (WebP/JPEG/PNG), стиснення через sharp
- Batch compress всіх зображень

#### 💻 Code Editor (тільки Dev mode)
- Дерево файлів проекту зліва (тільки текстові: html/css/js/php/json/svg тощо)
- **Monaco Editor** з підсвіткою синтаксису (автодетекція по розширенню)
- **Emmet** підтримка для HTML і CSS (`div.container>ul>li*3` → Tab → розгортається)
- Кнопка **"Open Preview ↗"** — відкриває превʼю у новій вкладці
- Після **Save** — превʼю у тій вкладці автоматично перезавантажується через `BroadcastChannel`

### Крок 4 — PHP Integration (Dev)

| Поле | Приклад | Де використовується |
|---|---|---|
| Offer Name | `Quantum AI` | Hidden input `offer_name`, константа `OFFER_NAME` |
| Country Code | `DE` | Константа `COUNTRY_CODE` |
| Language Code | `de` | Атрибут `lang`, `getFormJSCss()`, `LANGUAGE_CODE` |

**PHP сніпети в `index.php`:**
```
Перед <!DOCTYPE>  → require_once global_new.php
Після <head>      → require_once google_event.php
Перед </head>     → echo getFormJSCss('de')
Перед </body>     → require_once offer_footer_script.php
Після <form>      → hidden input offer_name + require_once hidden_params.php
```

Всі ін'єкції **ідемпотентні** — повторний Apply не дублює сніпети.

Генерується `send.php`. Вкладка **"send.php Preview"** — фінальний вміст у реальному часі.

### Крок 5 — Build

Показує дерево файлів. **Download ZIP** — архів із рівнем стиснення 9. Виключаються файли з префіксом `_` та `raw/`.

## 8. Dev Access — перегляд сесії

URL: `/dev?dev=<sessionId>` (генерується кнопкою "Share Session" в Dev pipeline або відкривається зі списку сесій).

### Вкладки DevPanel

**📁 Files**
- Дерево файлів сесії
- Клік на файл → **Monaco Editor** з підсвіткою по типу файлу
- Кнопка **Save** — зберігає файл + оновлює `lastDevActivity` (баєр отримає нотифікацію)

**📋 Activity Log**
- Хронологічний лог всіх дій із сесією (upload, clean, save, push тощо)

**⚙️ Config**
- Збережена PHP-конфігурація (`_offer_config.json`)

### "▶ Start full pipeline" — запуск з оригіналів баєра

Кнопка в шапці DevPanel. Доступна якщо баєр завантажив оффер у Standard режимі (є `_originals/`).

**Що відбувається:**
1. Клонується `_originals/` (файли до auto-clean) в нову dev-сесію
2. Відкривається Dev pipeline, стартуючи з кроку **Clean**
3. В шапці pipeline — кнопка **"⬆ Push to buyer"**

**"⬆ Push to buyer"**
- Копіює поточні файли dev-сесії назад у сесію баєра
- Оновлює `lastDevActivity` → баєр бачить нотифікацію "Developer updated the session ↺"
- Не перезаписує `_originals/`, `_activity_log.json`, `_session_meta.json`, `_offer_config.json`

### Список сесій

`/dev?sessions=1` — таблиця всіх активних сесій: ID, дата, кількість файлів, статус конфігурації.

## 9. Архітектура сервера

```
index.js
├── Middleware: cors, express.json (50mb), express.urlencoded (50mb)
├── Static: /session-files/:sid/ → sessions/:sid/
├── Static: /widgets/ → widgets/
├── Routes:
│   ├── /api/upload    → upload.js
│   ├── /api/process   → process.js

│   ├── /api/widgets   → widgets.js
│   ├── /api/php       → php.js
│   ├── /api/build     → build.js

└── Cron: кожні 6г — видалення сесій старших 24г
```

**Preview iframe injection** (`content.js`):
- `buildEditorScript(sid)` генерує JS, який інжектується перед останнім `</body>` (через `lastIndexOf`)

- Відео: transparent overlay div блокує відтворення при кліку
- postMessage API між iframe та батьківською сторінкою: `ept-select`, `ept-img-select`, `ept-video-select`, `ept-catalog`, `ept-update`, `ept-highlight`, `ept-deselect`

## 11. Сервіси

### `folderScanner.js`
Рекурсивний пошук `index.html`. Вибирає за мінімальною глибиною, при рівній — максимальна кількість асетів поруч (бонус за `js/`, `css/`, `assets/`).

### `fileNormalizer.js`
Нормалізує структуру та оновлює всі посилання: `<link href>`, `<script src>`, `<img src>`, `srcset`, `data-src`, `url()` у CSS, `@import` (рекурсивно), `<video>`, `<meta og:image>`.

### `htmlProcessor.js`
- `extractScripts()` — класифікація по патернах (Tracking/Critical/Unknown)
- `removeScripts()` — видалення по індексах
- `injectSnippet()` — вставка в задану позицію
- `detectInsertionPoints()` — автовизначення точок вставки

### `htmlFormatter.js`
Prettier 3 (async). Parser: `html`, printWidth: 120, tabWidth: 2. При помилці повертає оригінал з `success: false`.

### `widgetInjector.js`
Копіює файли віджета в сесію, додає `<link>` у `<head>`, вставляє HTML + `<script>` у задану позицію.

### `activityLogger.js`
Записує всі дії в `_activity_log.json` сесії з timestamp та деталями.

### `sendPhpGenerator.js`
Генерує `send.php` з `templates/send.php.template`, підставляючи параметри офера.
